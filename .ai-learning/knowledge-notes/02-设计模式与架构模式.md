## 核心知识点

### GoF模式
- 选模式前先看**问题结构特征**：1:1 路由→策略，1:N 广播→观察者，链式传递/过滤→责任链
- **策略 + 工厂（注册表）** 是成对出现的：策略解决多实现，工厂解决路由（消除 if-else）
- 工厂内部可通过 `Map<String, Strategy>` + Spring 自动注入实现零 if-else 注册
- **适配器模式**：用于包装无法修改的第三方接口，使其适配自己的统一接口
- **入站路由 vs 出站策略**：入站请求路由由网关负责，服务内部调第三方的版本/策略选择由工厂内部负责
- **模板方法（Template Method）**：算法骨架固定（final），部分步骤由子类实现（abstract）
- **装饰器（Decorator）**：运行时动态叠加增强行为，**一定转发**原对象
- **代理（Proxy）**：控制对原对象的访问，**可拦截不调**原对象

### 模式识别速查表
| 问题结构特征 | 首选模式 |
|------------|---------|
| 1:1 路由，多实现选一个 | 策略 |
| **1:N 广播，一个事件触发多个动作** | **观察者** |
| 链式传递/过滤 | 责任链 |
| 步骤固定，部分可变 | 模板方法 |
| 不改原类，动态叠加行为 | 装饰器 |
| 控制访问，透明拦截 | 代理 |
| 构造复杂对象，参数多可选 | 生成器 |
| 包装改不了的第三方接口 | 适配器 |

### CQRS（命令查询职责分离）
- 核心：**模型分离**，非仅 DB 分离。写模型范式化，读模型反范式化（宽表/预聚合）零 JOIN
- **同步方式：事件驱动 / CDC（Canal），不能用主从复制**（主从复制=同结构，宽表=不同结构）
- 延迟处理用 **Read-your-own-writes**：客户端乐观展示 / 写后读主库 / Session级路由
- 适用：读写比悬殊、读写优化方向相反

### Event Sourcing（事件溯源）
- **事件不可变**（append-only），是 source of truth
- 状态是从事件推导的视图，**可随时重算**
- **Snapshot**：定期保存状态避免全量 replay，可从事件流重建（与冗余字段的关键区别）
- Bug 修复：修代码→删快照→从事件 replay→得正确状态

### DDD（领域驱动设计）
- **贫血模型**：实体只有 getter/setter，逻辑全在 Service → 难测试、难维护
- **充血模型**：实体有行为方法 → 问「能做什么」而非「有什么」
- 充血实体典型方法：状态流转（submit/approve/reject）、自校验（validateRules）、适用判断（isApplicableTo）、业务计算（calculateDiscount）、状态查询（isActive）
- **逻辑归属三层判断**：
  - 只依赖实体自身属性 → **实体内部**
  - 跨多个实体/聚合 → **领域服务（Domain Service）**——放**业务规则**
  - 调外部系统/管事务/发消息/编排流程 → **应用服务（Application Service）**——放**流程胶水**
- **领域服务 vs 应用服务判断口诀**：
  - 这段逻辑是**业务规则**（如冲突检查、金额计算）？→ 领域服务
  - 这段逻辑是**调外部、发事件、管事务、编排多步**？→ 应用服务
  - 调短信API、调支付API、发MQ、写审计日志 → 都是**应用服务**编排
- **跨聚合判断口诀**：「这个数据是谁的？」不是实体自己的数据 → 不在实体内部处理

### 六边形架构（Ports & Adapters）
- **依赖反转（DIP）**：领域层定义接口（Port），基础设施层提供实现（Adapter）
- 领域层依赖抽象接口，不依赖具体实现（MyBatis/MQ/HTTP）
- **入站适配器**：Controller、MQ Listener（外部请求→领域调用）
- **出站端口**：领域层声明的依赖接口（PromotionRepository、EventPublisher）
- **出站适配器**：MyBatisRepository、RabbitMQPublisher（实现出站端口）
- 收益：领域层可纯单测（mock 接口）、基础设施可替换、领域无框架注解

## 易混淆概念

| 概念A | 概念B | 关键区别 | 记忆锚点 |
|-------|-------|---------|---------|
| 责任链 | 观察者 | 1:1链式传递 vs 1:N广播 | 审批流=责任链，通知=观察者 |
| 策略模式 | 适配器模式 | 自己定义接口 vs 包装别人接口 | 自己写→策略，包别人→适配器 |
| 模板方法 | 装饰器 | 继承+编译时 vs 组合+运行时 | 固定流程=模板方法，动态加料=装饰器 |
| 装饰器 | 代理 | 增强(一定转发) vs 控制(可拦截) | 加密=装饰器，缓存=代理 |
| 模板方法 | Builder | 固定算法骨架 vs 构造复杂对象 | 执行流程=模板方法，组装产品=Builder |
| DB读写分离 | CQRS | 同结构分机器 vs 不同结构分模型 | 同表=DB分离，不同表=CQRS |
| 主从复制 | 事件/CDC同步 | 同结构物理复制 vs 不同结构逻辑转换 | 同结构→主从，不同结构→事件/CDC |
| 冗余字段 | ES Snapshot | 独立更新不可重建 vs 可从事件流重建 | 能从事件重算=Snapshot |
| 贫血模型 | 充血模型 | 实体只有getter vs 实体有行为 | 问「有什么」=贫血，问「能做什么」=充血 |
| 实体内部 | 领域服务 | 只依赖自身属性 vs 跨多个聚合 | 数据是自己的=实体，数据是别人的=领域服务 |
| **领域服务** | **应用服务** | **业务规则** vs **编排/事务/外部调用** | **业务逻辑=领域，流程胶水/调API=应用** |

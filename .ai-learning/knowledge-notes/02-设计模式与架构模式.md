## 核心知识点

### GoF模式
- 选模式前先看**问题结构特征**：1:1 路由→策略，1:N 广播→观察者，链式传递/过滤→责任链
- **策略 + 工厂（注册表）** 是成对出现的：策略解决多实现，工厂解决路由（消除 if-else）
- 工厂内部可通过 `Map<String, Strategy>` + Spring 自动注入实现零 if-else 注册
- **适配器模式**：用于包装无法修改的第三方接口，使其适配自己的统一接口；自己写的实现类直接实现统一接口即可
- **入站路由 vs 出站策略**：入站请求路由由网关负责，服务内部调第三方的版本/策略选择由工厂内部负责
- 灰度/多版本共存：version + channel 组合键 + 配置中心动态比例路由
- 观察者/发布-订阅：事件驱动，发布端 fire-and-forget，消费端分层（过滤层 + 发送层）
- **模板方法（Template Method）**：算法骨架固定（final 方法定顺序），部分步骤由子类实现（abstract）；通用步骤在基类，可变步骤留给子类
- **装饰器（Decorator）**：不修改原类，通过组合在运行时动态叠加行为；典型案例 Java InputStream 体系
- **模板方法 vs 装饰器**：模板方法 = 继承 + 编译时固定骨架；装饰器 = 组合 + 运行时动态增强

### 模式识别速查表
| 问题结构特征 | 首选模式 |
|------------|---------|
| 1:1 路由，多实现选一个 | 策略 |
| 1:N 广播，多方同时响应 | 观察者 |
| 链式传递/过滤 | 责任链 |
| 步骤固定，部分可变 | 模板方法 |
| 不改原类，动态叠加行为 | 装饰器 |
| 构造复杂对象，参数多可选 | 生成器 |
| 包装改不了的第三方接口 | 适配器 |
